This page covers creating a Fine-grained Personal Access Token (PAT) for GitHub. PATs have a rate limit of **5,000 API requests per hour**.

<Tip>For higher rate limits, see [Configure a GitHub App Installation](https://github.com/SpecterOps/bloodhound-docs/blob/main//openhound/collectors/github/configure-app).</Tip>

## Create a Fine-grained Personal Access Token


<Steps>
  <Step title="Navigate to Personal Access Tokens">
    Navigate to your GitHub **Profile** > **Settings**.
    <Frame>
      <img src="/images/extensions/github/pat/1_proile_settings.png" alt="GitHub profile settings menu"/>
    </Frame>
  </Step>

  <Step title="Open Developer Settings">
    Go to **Developer settings** (bottom left).
    <Frame>
      <img src="/images/extensions/github/pat/2_developer_settings.png" alt="Developer settings in GitHub profile"/>
    </Frame>
  </Step>

  <Step title="Go to Fine-grained Tokens">
    Navigate to **Personal access tokens** > **Fine-grained tokens**.
    <Frame>
      <img src="/images/extensions/github/pat/3_fine-grained_tokens.png" alt="Fine-grained tokens in Developer settings"/>
    </Frame>
  </Step>

  <Step title="Generate a new token">
    Click **Generate new token**.
    <Frame>
      <img src="/images/extensions/github/pat/4_generate_token.png" alt="Generate new fine-grained token page"/>
    </Frame>
  </Step>

  <Step title="Set resource owner">
    Set the **Resource owner** to your target organization.
  </Step>

  <Step title="Configure the access scope">
    Under **Repository access**, select **All repositories**.

    <Frame>
      <img src="/images/extensions/github/pat/5_all_repositories.png" alt="Select All repositories for token access"/>
    </Frame>
  </Step>
  <Step title="Configure the repository and org permissions">
    Configure the required [repository and organization permissions](https://github.com/SpecterOps/bloodhound-docs/blob/main//openhound/collectors/github/collect-data#required-permissions) to **Read-only**.

    <Frame>
      <img src="/images/extensions/github/pat/6_permissions.png" alt="Token permissions configuration"/>
    </Frame>
  </Step>

  <Step title="Generate the token">
      <Warning>You must save the token value (preferably in a password manager) at this point. You will not be able to recover it later.</Warning>

    Click **Generate token** and copy the token value.

    <Frame>
      <img src="/images/extensions/github/pat/7_save_pat.png" alt="Copy the generated token"/>
    </Frame>
  </Step>
</Steps>


<Tip>For organizations with SAML SSO enabled, you must [authorize the PAT for SSO access](https://docs.github.com/en/enterprise-cloud@latest/authentication/authenticating-with-saml-single-sign-on/authorizing-a-personal-access-token-for-use-with-saml-single-sign-on) after creating it.</Tip>

## Next Steps

After creating a token, proceed to [configure the collector](https://github.com/SpecterOps/bloodhound-docs/blob/main//openhound/collectors/github/collect-data) to start collection.
