Zones define hierarchical privilege levels in your environment based on a tiered administration model. The most common tiering model is [Microsoft's Enterprise Access Model](https://learn.microsoft.com/en-us/security/privileged-access-workstations/privileged-access-access-model). 

BloodHound uses zones to measure risk and detect violations. Each zone has a specific tier level (**Tier Zero** is the default and highest). You can create multiple zones (e.g., Tier One, Tier Two) to match your security model.

The **Zones** tab offers two views:

<Tabs>
  <Tab title="Summary View">
  The **Summary View** shows zone names, selector counts, member count, and their hierarchy relative to other zones (the top zone is most critical).
    <img
      src="/images/privzones/summary-view.png"
      alt="Privilege Zones summary view"
      style={{ width:"100%" }}
    />
  </Tab>
  <Tab title="Detail View">
  The **Detail View** lists every selector and member that each selector pulls into the associated zone.
    <img
      src="/images/privzones/detail-view.png"
      alt="Privilege Zones detail view"
      style={{ width:"100%" }}
    />
  </Tab>
</Tabs>

### Create a zone

Creating a zone involves configuring the zone details and defining a selector.

<SelectorWarning/>

<Steps>
  <Step title="Open the Privilege Zones page">
  In the left menu, click **Privilege Zones** > **Zones** > **Create Zone**.
  </Step>

  <Step title="Configure initial zone details">
  Enter all relevant information about the zone:

  | Field                 | Required?    | Description                                                                                                          |
  |-----------------------|:------------:|----------------------------------------------------------------------------------------------------------------------|
  | Name                  | Yes          | A unique name for the zone (e.g., Server Tier)                                                                       |
  | Description           | No           | A brief description of the zone's purpose and scope (e.g., PCI assets)                                               |
  | Require Certification | No           | **_[BHE Only]_** An option to mandate [certification](#certification) for all members within this zone                                                  |
  | Apply Custom Glyph    | No           | **_[BHE Only]_** Option to apply a custom glyph to visually distinguish members of this zone in the **Explore** page |

    <img
      src="/images/privzones/create-zone.png"
      alt="Configure a new privilege zone"
      style={{ width:"70%" }}
    />

  </Step>

  <Step title="Define a selector">
  Click **Define Selector** to save your new Privilege Zone and continue on to define the objects to include in the zone.

  <SelectorWarning/>

  When defining a selector during the zone creation process, provide the following information:

  | Field                  | Required?          | Description                                                                                   |
  |------------------------|:------------------:|-----------------------------------------------------------------------------------------------|
  | Name                   | Yes                | A unique name for the selector (e.g., PCI Assets)                                             |
  | Description            | No                 | A brief description of the selector's purpose and scope (e.g., PCI assets)                    |
  | Selector Type          | Yes                | The type of selector to use (e.g., Object ID or Cypher)                                       |
  | Automatic Certification| No                 | **_[BHE Only]_** An option to choose how BloodHound [certifies](#certification) new objects                    |

    <img
      src="/images/privzones/define-zone-selector.png"
      alt="Define a selector for the privilege zone"
      style={{ width:"100%" }}
    />
  </Step>
    <Step title="Complete zone creation">
    Click **Save** to finish creating the zone.
    </Step>
</Steps>

### Edit or delete a zone

To edit or delete an existing zone, follow these steps:

<Note>You cannot delete the default **Tier Zero** zone, but you can modify its properties. See [Modify Tier Zero](https://github.com/SpecterOps/bloodhound-docs/blob/main//get-started/security-boundaries/modifying-tier-zero) for more information.</Note>

<Steps>
  <Step title="Select a zone">
   Navigate to the **Privilege Zones** page, select the zone you want to edit or delete, and click **Edit**.
  </Step>

  <Step title="Edit or delete a zone">
    Choose one of the following actions:

    <Tabs>
      <Tab title="Edit a zone" icon="edit">
      To edit the zone:

      1. Modify one of the available fields.

         For example, you can modify the zone's name, description, [certification](#certification) requirements, and custom glyph.

       1. With the exception of the default **Tier Zero** zone, you can also change the zone's hierarchical position by using the vertical grip control (<Icon icon="grip-vertical" iconType="solid" />) in the **Zone Order** panel to reorder it.

          <img
            src="/images/privzones/reorder-zone.png"
            alt="Reorder privilege zones"
            style={{ width:"70%" }}
           />

       1. Click **Save Edits** to apply your changes.
      </Tab>

      <Tab title="Delete a zone" icon="trash-can">
      To delete the zone:

      1. Click <Icon icon="trash-can" /> **Delete Zone** at the bottom of the page.
      1. Confirm your action in the dialog.

          <img
            src="/images/privzones/confirm-zone-delete.png"
            alt="Confirm deletion of a custom privilege zone"
            style={{ width:"70%" }}
           />

      1. Click **Confirm** to delete the zone.
      </Tab>
    </Tabs>
  </Step>
</Steps>
