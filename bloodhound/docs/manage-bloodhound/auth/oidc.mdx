You can configure multiple SSO providers within your tenant if necessary. This page provides the overall steps to configure an OIDC provider within BloodHound.

<Note>
Entra ID is not currently supported with BloodHound's OIDC implementation. Use the [SAML configuration](https://github.com/SpecterOps/bloodhound-docs/blob/main//manage-bloodhound/auth/saml-entra-id) instead.
</Note>

## Order of Operations

You must configure OIDC in BloodHound in the following order:

1.  Determine the Identity Provider (IDP) name you will use for the OIDC configuration.

    The same value must be configured in both the IDP and BloodHound. The BloodHound callback URL will include this value.

1.  Configure the IDP for BloodHound.

    See the [Configure Okta](https://github.com/SpecterOps/bloodhound-docs/blob/main//manage-bloodhound/auth/oidc-okta) guide for configuring Okta as your OIDC provider.

1.  Create the OIDC configuration in BloodHound.

1.  Create new users or modify existing users using the UI or via the newly created OIDC provider.
