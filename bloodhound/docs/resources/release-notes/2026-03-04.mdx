  ## Table Column Resizing
    
  {/*BED-7355, BED-7055*/} Resize table columns to review data more comfortably.
    
    BloodHound now allows you to adjust column widths in the [table layout](https://github.com/SpecterOps/bloodhound-docs/blob/main//analyze-data/explore/search#table) graph visualization so you can view data on smaller screens without changing your window size.

  ## Meta Node Information in Entity Panel

  <img
    src="/assets/enterprise-edition-pill-tag.svg"
    alt="BloodHound Enterprise logo"
    style={{ width: "25%" }}
  />

  Review Meta node information without encountering errors.
  
  BloodHound Enterprise now displays [Meta](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/nodes/meta) node information in the **Entity Panel** for more consistent context. You may encounter these nodes in the graph after running some [Cypher queries](https://github.com/SpecterOps/bloodhound-docs/blob/main//analyze-data/explore/cypher-search).
  
  Meta nodes represent related-node groupings rather than directly collected objects. BloodHound Enterprise uses Meta nodes to map attack paths across zones and environments to enrich the graph.

<Update label="BloodHound" description="Enhancement" tags={["Explore"]}>

   ## Improved Edge Information

  - {/*BED-7274*/} Review accurate, complete edge guidance in the **Entity Panel** for both [AZOwner](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-owner) and [AZOwns](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-owns) while you investigate the graph.

    BloodHound now displays missing help text for AZOwner and aligns AZOwns content in the **Entity Panel** with the edge reference documentation.

  - {/*BED-7105*/} Review clearer Entra ID role edge guidance in the **Entity Panel** while you analyze Azure attack paths.

    BloodHound now standardizes Azure role edge help text to use Entra ID terminology, improves relationship details for role assignment and activation paths, and adds official Microsoft references for key role edges:

    - [AZAppAdmin](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-app-admin)
    - [AZCloudAppAdmin](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-cloud-app-admin)
    - [AZGlobalAdmin](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-global-admin)
    - [AZHasRole](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-has-role)
    - [AZPrivilegedAuthAdmin](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-privileged-auth-admin)
    - [AZRoleApprover](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-role-approver)
    - [AZRoleEligible](https://github.com/SpecterOps/bloodhound-docs/blob/main//resources/edges/az-role-eligible)

  - {/*BED-7479*/} Review valid node relationships against the new [`valid_edges.json`](https://github.com/SpecterOps/BloodHound/blob/main/schemas/valid_edges.json) schema.

    BloodHound now includes a reference schema that defines valid source-target node relationships and supported edge types. This schema is intended to help improve understanding of graph data for developers and advanced users working with the BloodHound graph structure.

</Update>

  - {/*BED-6661*/} Resolved an issue where selecting the <Icon icon="globe" /> (globe) icon in the graph on the **Attack Paths** page displayed Tier Zero objects in the **Zone Membership Panel** after selecting a different zone in the filter dropdown.   {/*BED-7273*/} Resolved an issue where the name of the _Computers with membership in Protected Users_ [prebuilt Cypher query](https://github.com/SpecterOps/bloodhound-docs/blob/main//analyze-data/explore/cypher-search) did not match its results.
