## General Information

The non-traversable [GH_AdminTo](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/github/edges/gh_adminto) edge represents a role's full administrative access to the repository. Admin is the highest built-in repository role and grants control over all repository settings, including dangerous operations like deleting the repository or modifying its visibility. Admin access bypasses most protections including branch protection rules, unless `enforce_admins` is explicitly enabled on the branch protection rule. This edge is a key permission in the computed branch access model and is a high-value target in attack path analysis.

```mermaid
graph LR
    user1("GH_User alice")
    adminRole("GH_RepoRole GitHound\admin")
    repo("GH_Repository GitHound")
    orgOwners("GH_OrgRole SpecterOps\Owners")
    allRepoAdmin("GH_RepoRole SpecterOps\all_repo_admin")
    user1 -- GH_HasRole --> adminRole
    adminRole -- GH_AdminTo --> repo
    orgOwners -- GH_HasBaseRole --> allRepoAdmin
    allRepoAdmin -- GH_AdminTo --> repo
```
