### Optional Schemas

If your GitHub environment uses SCIM, upload the [bh-scim-extension.json](https://github.com/SpecterOps/bloodhound-scim-extension/blob/main/bh-scim-extension.json) schema as well. This schema provides a shared model for provisioned users and groups across cloud identity providers and applications.

If GitHub is connected to other BloodHound-supported data sources in your environment, such as Okta, upload the corresponding schema too, or contact your Enterprise account team. Doing so ensures those cross-platform relationships are modeled correctly in BloodHound.

## Import Cypher Queries

The GitHub extension includes [Cypher queries](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/github/queries) to help identify attack paths and misconfigurations in GitHub environments. Import the GitHub query JSON files into BloodHound using the [query import workflow](https://github.com/SpecterOps/bloodhound-docs/blob/main//analyze-data/explore/cypher-search#import-and-export).

You can then run the imported queries on the **Explore** page.

<Note>Cypher queries that reference node or edge kinds not present in the database will fail without the extension schema.</Note>

## Collect and Upload GitHub Data

There are currently two ways to collect GitHub data for this extension:

- [OpenHound GitHub collector](https://github.com/SpecterOps/bloodhound-docs/blob/main//openhound/collectors/github/overview): The SpecterOps-supported collector for GitHub. Use this if you want the current documented and supported collection path.
- [GitHound collector](https://github.com/SpecterOps/GitHound): An alternative collector that can also produce data for the GitHub extension.

<Tip>Prefer the supported [OpenHound GitHub collector](https://github.com/SpecterOps/bloodhound-docs/blob/main//openhound/collectors/github/overview).</Tip>

## Configure Privilege Zones

Use the [Privilege Zone rules](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/github/privilege-zone-rules) to classify high-value GitHub assets.

The extension also includes specialized (Privilege Zone) queries for [creating Cypher-based Privilege Zone rules](https://github.com/SpecterOps/bloodhound-docs/blob/main//analyze-data/privilege-zones/rules#cypher).

Read the [Tier Zero Classification](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/github/tier-zero) page to understand the rationale behind the Tier Zero Privilege Zone rules.
