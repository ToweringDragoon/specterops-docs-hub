GitHub App installations provide a higher API rate limit of **15,000 requests per hour** compared to **5,000** for Personal Access Tokens(PATs).

<Note>This is the recommended authentication method for organizations with 500+ repositories.</Note>

## Install the GitHub App

You can either install the pre-configured SpecterOps GitHub App (recommended) or create your own.

<Tabs>
  <Tab title="Install the SpecterOps GitHub App">

    The SpecterOps GitHub App comes pre-configured with the minimum required permissions.

    1. Contact your SpecterOps representative for the GitHub App installation link.
    1. Select the GitHub Organization for installation.
    1. Review the requested permissions and click **Install**.
    1. Select **All repositories** or choose specific repositories.
    1. Click **Install** to complete the installation.
  </Tab>
  <Tab title="Create Your Own GitHub App">
    Follow the steps below to create and configure your own GitHub App.

    If you prefer to host your own GitHub App:

    1. Navigate to your **Organization Settings** > **Developer settings** > **GitHub Apps**.
    1. Click **New GitHub App**.
    1. Configure the app:
        - **GitHub App name**: Choose a unique name (e.g., `YourOrg-GitHound`)
        - **Homepage URL**: Your organization's URL
        - **Webhook**: Uncheck "Active" (not needed for collection)

    1. Set all required [repository and organization permissions](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/githound/collect-data#required-permissions) to **Read-only**.

    1. Under "Where can this GitHub App be installed?", select **Only on this account**.
    1. Click **Create GitHub App**.
    1. Scroll down and click **Generate a private key** — save the downloaded `.pem` file securely.
    1. Note the **App ID** and **Client ID** displayed on the app settings page.
    1. Click **Install App** in the left sidebar to install the app to your organization.
  </Tab>
</Tabs>

## Get the Installation Details

1. Navigate to your **Organization Settings** > **Third-party Access** > **GitHub Apps**.
1. Find the GitHound App and click **Configure**.

    <Frame>
      <img src="/images/extensions/githound/app-installation/app-installation-page.png" alt="GitHub App installation page showing permissions and repository access"/>
    </Frame>

1. The numeric value in the URL is the **App Installation ID**.
1. Click **App settings** to access the **Client ID** and **App ID**.

    <Frame>
      <img src="/images/extensions/githound/app-installation/app-installation-details.png" alt="GitHub App settings showing Client ID and App ID"/>
    </Frame>

## Security Considerations

- Store the `.pem` private key securely and never commit it to version control.
- Rotate keys periodically and revoke old ones.
- Only install the app on organizations that require collection.
- Periodically verify the app has only the minimum required permissions.

## Next Steps

After installing and configuring the GitHub App, proceed to [Running the Collector](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/githound/collect-data) to start collection.
