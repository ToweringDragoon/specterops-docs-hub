This page covers creating a Fine-grained Personal Access Token (PAT) for GitHound. PATs have a rate limit of **5,000 API requests per hour**.

<Tip>For higher rate limits, see [Configure a GitHub App Installation](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/githound/configure-app).</Tip>

## Create a Fine-grained Personal Access Token

1. Navigate to your GitHub **Profile** > **Settings**.

    <Frame>
      <img src="/images/extensions/githound/pat/1_proile_settings.png" alt="GitHub profile settings menu"/>
    </Frame>

1. Go to **Developer settings** (bottom left).

    <Frame>
      <img src="/images/extensions/githound/pat/2_developer_settings.png" alt="Developer settings in GitHub profile"/>
    </Frame>

1. Navigate to **Personal access tokens** > **Fine-grained tokens**.

    <Frame>
      <img src="/images/extensions/githound/pat/3_fine-grained_tokens.png" alt="Fine-grained tokens in Developer settings"/>
    </Frame>

1. Click **Generate new token**.

    <Frame>
      <img src="/images/extensions/githound/pat/4_generate_token.png" alt="Generate new fine-grained token page"/>
    </Frame>

1. Set the **Resource owner** to your target organization.
1. Under **Repository access**, select **All repositories**.

    <Frame>
      <img src="/images/extensions/githound/pat/5_all_repositories.png" alt="Select All repositories for token access"/>
    </Frame>

1. Configure the required [repository and organization permissions](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/githound/collect-data#required-permissions) to **Read-only**.

    <Frame>
      <img src="/images/extensions/githound/pat/6_permissions.png" alt="Token permissions configuration"/>
    </Frame>

1. Click **Generate token** and copy the token value.

    <Frame>
      <img src="/images/extensions/githound/pat/7_save_pat.png" alt="Copy the generated token"/>
    </Frame>

    <Warning>You must save the token value (preferably in a password manager) at this point. You will not be able to recover it later.</Warning>

    <Tip>For organizations with SAML SSO enabled, you must [authorize the PAT for SSO access](https://docs.github.com/en/enterprise-cloud@latest/authentication/authenticating-with-saml-single-sign-on/authorizing-a-personal-access-token-for-use-with-saml-single-sign-on) after creating it.</Tip>

## Next Steps

After creating a token, proceed to [Running the Collector](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/githound/collect-data) to start collection.
