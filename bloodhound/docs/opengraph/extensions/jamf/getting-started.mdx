### Optional Schemas

If your Jamf environment is connected to other BloodHound-supported data sources in your environment, such as Okta, upload the corresponding schema too, or contact your Enterprise account team. Doing so ensures those cross-platform relationships are modeled correctly in BloodHound.

## Import Cypher Queries

The Jamf extension includes [Cypher queries](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/jamf/queries) to help identify attack paths and misconfigurations in Jamf Pro environments. Import the Jamf query JSON files into BloodHound using the [query import workflow](https://github.com/SpecterOps/bloodhound-docs/blob/main//analyze-data/explore/cypher-search#import-and-export).

You can then run the imported queries on the **Explore** page.

<Note>Cypher queries that reference node or edge kinds not present in the database will fail without the extension schema.</Note>

## Collect and Upload Jamf Data

There are currently two ways to collect Jamf data for this extension:

- [OpenHound Jamf collector](https://github.com/SpecterOps/bloodhound-docs/blob/main//openhound/collectors/jamf/overview): The SpecterOps-supported collector for Jamf Pro. Use this if you want the current documented and supported collection path.
- [JamfHound collector](https://github.com/SpecterOps/JamfHound): An alternative collector that can also produce data for the Jamf extension.

<Tip>Prefer the supported [OpenHound Jamf collector](https://github.com/SpecterOps/bloodhound-docs/blob/main//openhound/collectors/jamf/overview).</Tip>

## Configure Privilege Zones

Use the [Privilege Zone rules](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/jamf/privilege-zone-rules) to classify high-value Jamf assets.

The extension also includes specialized (Privilege Zone) queries for [creating Cypher-based Privilege Zone rules](https://github.com/SpecterOps/bloodhound-docs/blob/main//analyze-data/privilege-zones/rules#cypher).
