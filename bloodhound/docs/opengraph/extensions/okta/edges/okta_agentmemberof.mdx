## Edge Schema

- Source: [Okta_Agent](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/okta/nodes/okta_agent)
- Destination: [Okta_AgentPool](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/okta/nodes/okta_agentpool)
- Traversable: ✅

## General Information

Okta_AgentMemberOf edges represent membership of an [Okta_Agent](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/okta/nodes/okta_agent) in an [Okta_AgentPool](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/okta/nodes/okta_agentpool).

Active Directory Agent Pools and their agents can be visualized in BloodHound as follows:


```mermaid
graph LR
    ap1("Okta_AgentPool contoso.com")
    ap2("Okta_AgentPool adatum.com")
    a1("Okta_Agent CONTOSO-SRV1")
    a2("Okta_Agent CONTOSO-SRV2")
    a3("Okta_Agent ADATUM-SRV1")
    a1 -- Okta_AgentMemberOf --> ap1
    a2 -- Okta_AgentMemberOf --> ap1
    a3 -- Okta_AgentMemberOf --> ap2
```

<Info>
Traversable edges between [Okta_AgentPool](https://github.com/SpecterOps/bloodhound-docs/blob/main//opengraph/extensions/okta/nodes/okta_agentpool) and AD Domain nodes are not modeled in the current version of the Okta BloodHound extension. Support for this is planned for a future release.
</Info>
